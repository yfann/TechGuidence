
# 网络

+ 二层设备(L2)
    - 物理层+链路层
    - 只能识别mac地址
    - 网桥,交换机
    - 连接以太网(局域网)

+ 三层设备(L3)
    - 物理层，链路层，网络层
    - 能识别mac地址,ip地址
    - 路由器，三层交换机,ip网关

+ 四层交换(L4)
    + OSI中第四层为传输层
    + 传输层负责端对端通信
    + TCP,UDP
    + 可以建立连接
        + 通过firewall
            +  L4 connections from A to B, connections at L3, A to B and B to A, must be allowed.

+ 广播域（MAC全为1能传到的范围）
    + 广播帧会传播到网络中的每台电脑
    + ARP广播
    + DHCP广播
    + 广播帧不会被switch过滤,switch收到后会flooding
    + 路由器可隔离冲突域和广播域
    
+ 冲突域
    + 设备共享同一网段
    + 同网段两个以上设备同时通讯时可能冲突
    + 交换机可隔离网段
        + 每个端口为新的网段

+ VLAN(虚拟局域网)
    + 分割广播域
        + 二层交换机只能构建单一广播域
        + 路由器的网络接口(LAN)为单位分割

+ 内网IP(Private IP)
    + 内网通过IP Sharing访问Internet
    + 内网计算机可以向internet上的计算机发请求，internet上的计算机不能向内网计算机发请求
    + 内网IP一般: 10.X.X.X, 172.16.X.X-172.31.X.X, 192.168.X.X
    + A private IP address is the address your network router assigns to your device

+ 公网IP(Public IP)
    + 静态公网IP
    + 动态公网IP
        + 路由器连接方式为pppoe,则是动态公网IP
    + 可以访问Internet,通常ISP提供，分配到router

+ CIDR
    + 205.123.196.183/25
        + /后表示前多少位表示网络

+ DHCP(动态主机配置协议)
    + provide dynamic IP

+ multiplexing(多路传输)
    + 网络上不同会话片段交错进行的过程
    + segmentation(分割)
    
+ protocal stack(协议栈)
    + OSI模型(七层)

+ Protocol Data Unit(PDU)
    + 数据片在各层中的结构
    + Data(应用层PDU)
    + Segment(传输层PDU)
        + 包含源MAC和目的MAC
    + Packet(网络层PDU)
    + Frame(链路层PDU)
    + Bits(物理层PDU)

+ 访问网络资源需要两种地址
    + 网络层地址
        + IP
            + 发送到相应网络和主机
            + DNS
    + 数据链路层地址
        + MAC(网卡地址)
            + 同一网络中，数据链路帧从一个网络接口发到另一个网络接口
            + 以太网LAN和无线网LAN介质不同，分别有自己的链路协议
        + 发送方主机使用 地址解析协议ARP(Address Resolution Protocol)检查本地网络所有MAC
            + IP(ARP request)---->MAC(ARP response)
                + 同一个网络中
            + 发送方主机在整个LAN中发送ARP请求（广播）
                + LAN上的每个设备会检查 ARP请求，如果IP相同则会发ARP响应
                    + ARP响应包含ARP请求中IP地址相对于的MAC地址

+ ingress
    + frame通过特定端口进入设备

+ egress
    + frame通过特定端口离开设备

+ 路由器
    + 可以分割广播域，冲突域
    + LAN接口有限
    + 主要功能是网络间传输数据



## 网关（gateway）

+ 协议转换
+ 默认网关
    + 同网络上的路由器IP地址
    + 跨网访问时经过网关
    + 连接本地网与公网
+ 一般指代路由器

## 集线器
+ 向所有端口广播数据
+ 接收方网卡根据MAC判断是否要丢弃包
+ 如果多台电脑同时发数据，则集线器不可用，数据会碰撞

## 交换机(switch)

+ 根据MAC转发包

+ 根据MAC地址表(端口<-->连接方MAC),把数据包发到对应的端口
    + 如果数据包的接收方MAC和MAC地址表中的MAC匹配，则数据包发往对应端口
    + 如果MAC地址表中找不到对应的记录，则除了接收数据包的端口外向其他端口广播数据
    + 如果数据包的接收方MAC是个广播MAC(FF:FF:FF:FF:FF:FF:),则向其他端口广播数据

+ 全双工模式
    + 可以同时发送接收数据
    + 数据不会碰撞

+ 分割冲突域,访问LAN，通过VLAN实现

+ 接收Frame并转发到目的地址

+ MAC地址表
    + 动态变化
    + 学习，记录端口与设备关系
    + switch port -> device MAC
        + 端口与连接方MAC的映射表
    + 交换机互联，一个端口会对应多个MAC

+ 交换机学习原地址，基于目的地址转发
    + Frame进入switch后，switch把Frame的source MAC及ingress(进入端口)记录到MAC地址表
    + 转发Frame时在MAC地址表中比较目的MAC
        + 如果存在,则在对应的端口转发Frame
        + 如果没有,则flooding
            + 向除ingress端口外所有端口发送Frame

+ store and forward(存储转发)
    + 差错控制(frame check sequence,FCS)
    + 自动缓存
        + 支持不同速率以太网混合连接


+ cut-through(直通交换)
    + 快速帧转发
        + 读完header就可以转发，不必等待帧全部进入端口

+ Fragment Free(无水平转发)

+ SAT(source address table)
    + 按照Frame的目的MAC,转发到合适端口


## VLAN(virtual local area network)

+ 与位置无关的逻辑端口
    + 一个switch可以有多个VLAN
    + 不同switch上主机可以在同一个VLAN
+ 不同VLAN连接必须通过路由器
+ 独立的三层网络
+ 隔离广播域
+ SAT
    + 记录: VLAN端口->MAC地址

+ trunk
    + 连接switch
    + 传递VLAN信息
    + 实现同一VLAN不同switch的通信
    + VLAN trunk不属于特定VLAN,而是交换机与路由器间多个VLAN的通道
    + VLAN tag
        + VID
        + CFI

+ `show mac address table`
+ `show vlan`

## 路由器
+ 路由器的各端口都有MAC,IP

+ 根据IP转发包

+ 根据路由表(接收方IP与端口映射)
    + 数据包中目的地IP与路由表中记录的IP比较，忽略主机部分
    + tips
        + 直连路径
        + 远端路径
        + 动态路由协议
    + 维护
        + 手动
        + 路由协议
    + 路由表中子网掩码为`0.0.0.0`的条目时默认路由
        + 也是默认网关
        + 找不到匹配路由时会发往默认路由

+ 数据包的流向
    + 接收和端口MAC匹配的数据包
        + 丢弃MAC头
    + 查询路由表确定输出端口
        + 根据数据包中的IP查找路由表
            + 如果查不到则丢弃数据包，发ICMP响应
    + IP头中的TTL
        + 每经过一个路由时-1
        + 为0时丢弃包
        + 默认64或128
    + 如果端口的MTU小于包的长度，数据包需要拆分
    + 设置接收方IP,MAC
        + 如果路由表网关列不为空，则为下一个转发目标
            + 为空，则数据包的接收方IP为下一个转发目标
        + 通过ARP,找到MAC地址

+ 不同网络间传输报文

+ process
    1. 从Frame中解析出三层报文(移除Frame头尾)
    2. 查看目的IP，路由表中查找最佳路由
    3. 找到最佳路径后，三层报文重新封装为二层Frame,转发到输出端口
        + IP地址不变
        + 二层数据链路地址会改变

+ 下一跳

+ 静态路由
    + 手动配置

+ 动态路由
    + 路由器直接通信，利用收到的信息更新路由表
+ `show ip route`

+ 路由聚合
    + 多个子网合并成一个子网

+ 端口类型
    + 以太网
    + ADSL

+ NAT(地址转换)
    + 内网地址(私有地址)
        + 不同内网可以有相同的IP（不同内网间不会相互通信）
        + 10.0.0.0~... 172.16.0.0~ ... 192.169.0.0~....
            + 只用于内网，公有地址不会分配
    + 固定地址(公有地址)
    + 公司网络分两部分，一部分可以访问互联网，分配公有地址.另一部分内网，分配私有地址
    + 内网访问互联网是需要地址转换（更改数据包的IP,端口）
        + 转换设备会把私有地址转为公有地址，并分配端口号(区分不同主机)
            + 维护一张表，记录原始IP与公有地址和端口的关系
        + 互联网返回响应时，会把公有地址转回私有地址
            + 删除地址表中的记录
    + 互联网上的服务器如果想访问内网服务器
        + 需要在地址表中手动的添加内网IP与公网IP和port的映射关系
+ 包过滤(防火墙)
    + 根据MAC头，IP头，TCP头过滤
    

## private network(专用网络)
+ 规范
    + RFC 1918(IPV4)
    + RFC 4193(IPV6)
+ 专网IP无法直接连互联网
    + 需要使用NAT(Network Address Translator)或代理服务器(proxy server)
+ 与公网IP相比，私有IP免费，适合局域网使用
+ HTTP/2 for gRPC
    + long lived connection

## TCP

+ 可靠性

+ 数据流控

+ 滑动窗口确认机制
    + 将独立的字节当做流来处理
        + sequence number
        + 数据流分段,限制分段的字节数,确认发送的分段

## 网络性能

+ 大型广播域分段提高网络性能
    + 设备在广播域产生大量的广播以及多播帧时就会与数据流竞争带宽
            + ARP
            + DHCP
            + STP
    + 路由器分割广播域
    + 二层交换机创建VLAN减少广播域

## socket(套接字)

+ 特殊的I/O接口，有文件描述符
+ 操作系统内核中的一个数据结构
    + 协议、本地地址、本地端口、远程地址、远程端口
        + 用来建立connect,本地socket和远程socket
+ 进程间通信的门户
+ 端口
    + 一台电脑上有多个进程，端口来标记和哪个进程通信
    + 0~1024(公共)
+ 类型
    + 流式socket（SOCK_STREAM）
        + TCP
    + 数据报socket（SOCK_DGRAM）
        + UDP
    + 原始socket （SOCK_RAW）
        + IP/ICMP
+ `netstat` 查看本机socket
    + 每一行一个socket

## 协议栈

+ TCP
    + 建立connect

+ IP模块
    + 添加IP头部(包含接收方和发送方的IP地址)
        + 接收方的IP
            + 来着应用
        + 发送方的IP
             + 根据路由表判断，填写网卡(inteface)的地址
                + 发送方IP与路由表中Destination IP比较
                    + 如果网络部分相同，择取对应interface的地址作为发送方IP
                + 路由表中的Gateway(路由器IP)
                    + interface->gateway
    + 添加MAC头部(包含MAC地址)
        + ARP


## ADSL

+ 用户->接入网->网络运营商

+ 接入网
    + ADSL
    + 专线
    + FTTH(光纤)

+ 互联网接入路由器(BAS) 
    + 互联网接入路由器 -> ADSL Modem
        + 添加 MAC,PPP,PPPoE头
    + 用户拨号上网->BAS
        + 使用PPP实现认证
            + 输入用户名密码认证
    + ADSL,FTTH
        + 用户直连BAS
            + 不需要认证
        + PPP(PPPoE)

+ ADSL Modem
    + 将包拆成ATM信元
    + 转成电信号，传到分离器
        + 分离器，分离电话和ADSL信号

## 防火墙

+ 包含大量包过滤规则
    + 检查包的头
        + MAC,IP,TCP,UDP...
    + 不检查包的内容

+ 规则简单，不需要日志可以用有包过滤功能的路由器当防火墙

## forward proxy（正向代理）

+ 缓存服务器方在客户端
+ 提供防火墙保护客户端
    + 可以检查内容
+ 客户端发往代理的请求，Get 后面是完整的网址

## reverse proxy(反向代理)

+ url中路径部分指向不同服务器
+ DNS解析引导
    + 客户端通过域名访问反向代理，知道反向代理的存在，反向代理再转发到服务器

## transparent proxy（透明代理）

+ 根据包中的接收方IP转发给服务器

+ 客户端只知道服务器，向服务器发请求，透明代理会在请求线路上拦截消息

## 缓存服务器

+ 在服务端

+ 在客户端
    + 减少网络流量

+ 在互联网端(CDSP 内容分发服务)
    + 服务商可以更新缓存内容

+ 通过DNS,重定向找到离客户端最近的缓存服务器

## 服务器

+ 端口识别socket,不同socket使用不同端口

+ 应用和协议栈用描述符指代socket

+ 接收请求模块
    + socket一直等待
        + accept后建立连接
            + 复制套接字
                + 端口也复制(使用发送方IP,port和接收方的IP,port来识别socket)
            + 移交给处理模块
            + 继续等待其他请求

+ 一个模块处理请求
    + 一个线程处理一个请求

+ 连接的建立需要有一方的等待
    + 等待的一方是服务器

## tips

+ 数据包流向
    + 浏览器(socket)->操作系统.协议栈(TCP,UPD,IP)->网卡(interface)->集线器->交换机->路由器---接入网---->网络运营商(POP->NOC)->服务器(防火墙->负载均衡)
        + 浏览器(解析器)->DNS服务器
            + 得到IP

+ MTU(Maximum Transmission Unit)

+ MSS(Maximum Segment Size)

+ IP(路由器)负责将包送达通信对象这一整体过程，包传到下一个路由器是以太网(交换机)来负责的

+ PPP(Point to Point Protocol)
    + 电话线，ISDN等通信线路使用的协议
    + 集成用户认证，数据压缩，加密等功能

+ 隧道
    + 套接字之间建立的TCP连接
    + BAS---隧道--->运营商路由器
    + 实现方式
        + TCP
        + 基于封装的隧道

+ POP(Point of Presence)接入点
    + 接入网连接的运营商的设备

+ NOC(Network operation center)

+ BGP(border gateway protocal)
    + 不同运营商之间交互路由信息
