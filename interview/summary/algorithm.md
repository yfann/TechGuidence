# algorithm

## haspmap
+ 缓存中间结果，避免重复计算
+ 用于：
    + two sum

## 单调栈
+ 对每个元素，找它左右两侧最近的某种元素
    + 下一个更大/更小
    + 元素连续

+ 要素
    + 栈里一般存索引而不是值
    + 单调递减栈/单调递增栈
    + 当新元素“破坏了栈的单调性”时出栈

+ O(n)
    + 每个元素只入栈一次，出栈一次

## binary heap
+ 你可以在 O(log n) 时间内插入元素，并在 O(log n) 时间内取出当前优先级最高的元素。因此它适用于任何“每次都要快速选出最优/最小/最大项”的场景。

+ 适合用于
    + 只关心堆顶元素
    + 不需要查找任意元素，删除中间元素，更新已有元素优先级

+ 场景
    + Top-K / 第 K 大（小）元素
        + 从海量数据中找最大的 100 个数
        + O(n log k)
    + 多路有序合并（k-way merge）
    + 事件/任务调度
        + 谁最先到期，先处理谁
