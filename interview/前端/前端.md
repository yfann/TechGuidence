+ Reflow(回流) vs Repaint(重绘)
```js
// 重绘（Repaint）：重绘是指当元素样式发生改变，但不影响其布局的情况下，浏览器重新绘制元素的可视部分。例如，改变元素的颜色或背景色。重绘操作不会影响其他元素的位置和大小。

// 回流（Reflow）：回流是指当页面布局（DOM 结构）发生改变或者元素的位置、大小等属性发生变化时，浏览器重新计算元素的几何属性，并重新构建页面布局。回流操作会导致其他元素的位置和大小也发生变化，需要重新进行布局计算。回流操作比重绘操作开销更大，因为它涉及到页面的重新布局。


// 优化：
// 减少重绘操作：合理使用 CSS，避免过多的样式改变，尽量使用类名切换样式而非直接修改样式属性。

// 减少回流操作：避免频繁访问导致回流的属性（如 offsetTop、offsetLeft、clientWidth 等），批量修改样式或元素时使用 CSS 类名一次性应用。
// 另外，现代浏览器提供了一些工具和API（如 requestAnimationFrame、Transform、Flexbox 等）来优化重绘和回流操作，开发人员可以利用这些工具来提高页面的渲染性能
```

+ Event Delegation
```js
// 事件委托是一种通过将事件绑定到父元素而利用事件冒泡原理来处理子元素上的事件的技术。当一个事件发生在子元素上时，事件会一直冒泡到父元素，然后通过事件委托机制，父元素上的事件处理函数将被触发。事件委托的作用是减少事件处理函数的数量，提高性能和代码的简洁性。通过将事件绑定到父元素上，我们可以在父元素上处理多个子元素的事件，而不需要为每个子元素都绑定事件处理函数。这对于动态添加或删除子元素的情况特别有用，因为我们无需重复绑定和解绑事件，只需要在父元素上绑定一次即可
```

+ 闭包（Closure）
```js
// 闭包是指函数能够访问其词法作用域以外的变量的能力。简单来说，当一个函数可以记住并访问它所在的词法作用域时，就产生了一个闭包

// 封装私有变量：通过闭包，可以创建私有变量，只能通过特定的函数访问和修改。
// 模块化开发：通过使用闭包，可以创建具有私有状态和公共接口的模块。
// 延迟执行：可以使用闭包来延迟函数的执行，例如在定时器、事件处理程序或回调函数中。
// 循环中的问题：在循环中创建闭包时，可以解决使用 var 声明变量带来的作用域问题。

function outerFunction() {
  var outerVariable = 'I am outside!';

  function innerFunction() {
    console.log(outerVariable);
  }

  return innerFunction;
}

var closure = outerFunction();
closure();  // 输出：'I am outside!'
```

+ 什么是跨域请求（Cross-Origin Requests）
```js
// 跨域请求是指在浏览器中，通过 AJAX、Fetch API 或 WebSocket 等方式发送的 HTTP 请求，该请求的目标资源位于不同的域、子域、端口或协议下。出于安全原因，浏览器限制了跨域请求，防止恶意的网站获取到用户的数据或执行非法操作。

// JSONP（JSON with Padding）：利用 <script> 标签的跨域特性，通过动态创建 <script> 标签来加载一个位于其他域的 JavaScript 文件。服务器返回的 JavaScript 代码需要在响应中作为函数调用的参数，从而实现数据传递。

// CORS（Cross-Origin Resource Sharing）：CORS 是一种现代浏览器支持的跨域解决方案。通过在服务器端设置响应头，允许指定的源（域、协议、端口）访问资源。服务器可以设置 Access-Control-Allow-Origin 头来指定允许的源，还可以使用其他 CORS 相关头来控制不同类型的请求（如跨域请求中的预检请求 OPTIONS）。

// 代理服务器：可以设置一个位于同源的代理服务器，将跨域请求转发到目标服务器。浏览器直接请求同源的代理服务器，而代理服务器再转发请求到目标服务器，并将响应返回给浏览器。这样可以绕过浏览器的跨域限制。

// WebSocket：WebSocket 是一种支持跨域的双向通信协议。在建立 WebSocket 连接时，浏览器会发送一个初始的 HTTP 请求，通过服务器返回的响应头中的 Upgrade 字段切换到 WebSocket 协议，建立跨域的双向通信通道。
```

+ 前端什么时候会render
```js
// 初始加载：当用户首次访问网页时，浏览器会下载 HTML、CSS 和 JavaScript 文件，并解析执行这些文件以生成页面内容。这个过程就是初始的页面渲染。

// 数据变化：当页面中的数据发生变化时，通常需要更新对应的 DOM 元素来反映这些变化。例如，当用户在表单中输入内容时，或者通过 AJAX 请求获取到新的数据时，可以通过更新 DOM 元素来渲染这些变化。

// 事件触发：当用户与页面进行交互时，例如点击按钮、滚动页面或者输入文本等，可能会触发相应的事件处理函数。在事件处理函数中，可以根据需要修改相关的 DOM 元素，并触发重新渲染。

// 路由切换：在单页面应用（Single-Page Application，SPA）中，通过使用前端路由库（如 React Router、Vue Router 等），可以在不重新加载整个页面的情况下，根据 URL 的变化切换显示不同的内容。当路由发生切换时，新的组件会被加载并进行渲染。
```
