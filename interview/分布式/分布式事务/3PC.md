# 3PC
+ 协调者和参与者都引入了超时机制
+ 在2PC中插入一个准备阶段，保证了在最后提交阶段之前各个参与节点的状态是一致的

+ 阶段
    + CanCommit
    + PreCommit
        + 执行事务不提交
    + DoCommit

+ 优点
    + 3pc降低了阻塞范围，在等待超时后协调者或参与者会中断事务
    + 避免协调者单点问题，在docommit中协调者出现问题时，参与者会继续提交事务

+ 缺点
    + 数据不一致
        + 参与者在收到preCommit请求后等待doCommit时，如果协调者请求中断事务，而协调者无法与参与者通信，会导致参与者继续提交事务，造成数据不一致
