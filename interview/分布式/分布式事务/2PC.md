# 2PC
+ two-phase commit
    + 准备阶段
        + 协调组询问参与者是否提交
            +  收集反馈结果
    + 提交阶段
        + commit/rollback

+ 缺点
    + 性能问题
        + 参与节点事务为阻塞型
    + 可靠性
        + 参与者发生故障，任务失败
        + 协调者发生故障,参与者会一直阻塞下去
    + 数据一致性
        + 协调者commit后挂了，参与者同时也挂了
            + 没人知道事务是否提交
    + 复杂
        + 不适合高并发场景

+ 优点
    + 强一致
