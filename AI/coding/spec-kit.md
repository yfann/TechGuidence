# spec-kit
+ /constitution -> /specify -> /plan -> /tasks -> /implement

+ `/constitution <your input>`
    + 基本原则
    + 所有spec多会使用
    

 + `/specify <your input>`
    + 新的feature,会自动创建新branch去实现
    + what + why
   

+ `/clarify`
    + /specify之后执行
    + 让code agent检查时候还要未澄清的部分，未指定的部分
    

+ `/plan <your input>`
    + 具体要用的技术
    + how

+ `/tasks`  
    + 转化为可执行的逻辑


+ `/analyze`
    + /tasks之后执行
    + 查看plan之间是否有冲突


+ `/implement`
    + 调用code agent
